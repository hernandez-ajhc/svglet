# Contributing to svglet

Thank you for your interest in contributing to **svglet**! We welcome contributions from everyone.

## How to Contribute

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** to your local machine.
3. **Create a new branch** for your feature or bug fix: `git checkout -b my-new-feature`
4. **Make your changes**.
5. **Test your changes** by running the examples in `examples/`.
6. **Commit your changes**: `git commit -m 'Add some feature'`
7. **Push to the branch**: `git push origin my-new-feature`
8. **Open a Pull Request**.

## Project Structure

- `src/`: Source code modules.
- `dist/`: Compiled/minified output (if applicable, currently manual).
- `docs/`: Documentation files.
- `examples/`: Demo pages to test your changes.

## Coding Standards

- Use modern JavaScript (ES6+).
- Keep code clean and readable.
- Ensure no external dependencies are introduced.

## Reporting Bugs

Please open an issue on GitHub if you find a bug. Include steps to reproduce and any relevant browser console errors.
